使用帮助

快速了解 Diff确认平台 的核心功能与操作指南

新建项目 配置仓库 周版本管理 查看Diff 确认提交 缓存管理
1

新建项目

创建一个新项目来管理你的配表仓库

1

进入项目管理页面

点击顶部导航栏的 「项目列表」 进入主页，点击 「新建项目」 按钮。

2

填写项目信息

在弹出的表单中填写以下信息：

  • 项目代号：项目的唯一标识符，如 G119，用于URL路径
  • 项目名称：项目的可读名称，如「某某项目」
  • 项目描述：（可选）简要描述项目用途
3

提交创建

确认信息无误后点击「创建」按钮，项目即创建成功。创建后可在项目卡片中进一步配置仓库。

项目代号创建后不可修改，请谨慎填写。建议使用简短的英文+数字组合。
2

配置 SVN / Git 仓库

为项目添加代码仓库以追踪配表变更

1

进入项目详情

在项目列表中点击对应项目的 「管理」 按钮，进入项目管理页面。

2

添加仓库

点击 「添加仓库」 按钮，选择仓库类型并填写信息：

Git 仓库
  • 仓库名称：仓库显示名
  • 仓库URL：Git 远程地址（HTTPS 或 SSH）
  • 默认分支：如 main 或 master
  • 认证方式：用户名密码或SSH密钥
SVN 仓库
  • 仓库名称：仓库显示名
  • 仓库URL：SVN 仓库地址
  • 用户名：SVN 登录用户名
  • 密码：SVN 登录密码
3

同步提交记录

仓库添加成功后，系统会自动开始同步提交记录。你也可以手动点击 「同步」 按钮来获取最新提交。

Excel 配置表参数说明

表头行数 必填

指定 Excel 文件中表头占据的行数。系统根据此值区分表头和数据行，正确解析 Diff。例如表头占 3 行，则第 4 行开始为数据。

关键列

能够唯一标识一行数据的列号（从 1 开始），多个列号用英文逗号分隔，如 1,2,3。系统根据关键列匹配新旧版本中的同一行，精确计算行级和单元格级变更。若不填写，默认按行号顺序对齐。一般以ID作为关键列，如果默认第二列（即B列）为ID，可以设置关键列为 2 。

开启按 ID 确认

勾选后，Diff 页面支持按行（根据关键列 ID）逐条确认变更状态，适用于需要逐行审阅的精细化确认场景。需先正确配置「关键列」。

显示重复 ID 提示

勾选后，当 Excel 数据中存在关键列值重复的行时，Diff 页面会高亮提示重复 ID，帮助发现配表中的数据异常。

路径 / 文件类型正则

使用正则表达式过滤要追踪的文件路径和类型。只有匹配该正则的文件才会生成 Diff。例如 ^(client_data|server_data)/.*\.(xlsx|xls)+ 表示只追踪 client_data 或 server_data 目录下的 Excel 文件。

Log 正则

用正则表达式从提交日志（commit message）中提取关键信息。例如 序(\d+) 可从提交消息中提取序号，提取的内容会展示在提交列表中便于快速查看。

Log 过滤正则

使用正则表达式过滤不需要的提交记录。匹配该正则的提交将被自动隐藏，例如 ^(Merge branch).* 可过滤掉所有合并提交。

提交人过滤

指定需要过滤的提交人邮箱，多个邮箱用英文逗号分隔。匹配的提交人所提交的记录将被隐藏，适用于过滤自动化工具提交等场景。

重点表名

指定需要重点关注的 Excel 表名（Sheet 名或文件名），多个名称用英文逗号分隔。被标记为重点的表在 Diff 展示中会优先排列并高亮标识，便于快速关注关键配表变更。

以上「高级设置」中的字段（路径正则、Log 正则、Log 过滤正则、提交人过滤、重点表名）在添加仓库时默认折叠，点击「更多设置（可选）」展开即可配置。
请确保仓库地址可访问，认证信息正确。如果使用SSH方式，请确认服务器已配置对应的SSH密钥。

周版本管理

按周管理配表版本，支持合并多次提交的 Diff

1

创建周版本配置

在项目管理页面，点击 「周版本管理」 标签页，然后点击 「新增周版本」。

  • 版本名称：如「2025年第12周」
  • 起止日期：设定该周版本覆盖的时间范围
  • 关联仓库：选择要追踪的仓库
2

查看周版本 Diff

周版本创建后，系统会自动合并该时间范围内所有提交的 Diff。点击 「查看Diff」 可以看到整个周期的合并变更。

周版本 Diff 会合并所有单次提交的变更，只展示最终的新旧值差异，便于整体审阅。
4

查看 Excel Diff

对比 Excel 配表文件的变更内容

1

进入仓库提交列表

在项目管理页面，点击对应仓库的 「提交记录」，查看所有提交历史。

2

查看单次提交 Diff

点击某次提交记录的 「查看Diff」 按钮，进入 Diff 详情页面。

  • 页面顶部显示提交的基本信息（提交者、时间、消息）
  • 如果提交包含 Excel 文件变更，将以表格形式显示差异
  • 多个 Sheet 页通过标签页切换查看
3

确认状态管理

每条提交记录都有确认状态，可标记为「待确认」「已确认」「已拒绝」，方便团队协作追踪。

Diff 颜色与标识说明

行级别变更

1001 new_key 新增的值 true
新增行（整行绿色底色）

表示该行是新版本中新增的数据行，之前不存在。

502 old_key 已删除的值 false
删除行（整行红色底色）

表示该行在新版本中已被删除，只存在于旧版本中。

单元格级别变更

677 some_key 旧的值 新的值 true
修改的单元格

当某一行只有部分单元格发生变化时：

  • 红色背景 表示 旧值（修改前的值）
  • 绿色背景 表示 新值（修改后的值）
  • 未变化的单元格保持白色背景

行号标识

1001 新增行 502 删除行 677 修改行
行号颜色

行号列也会用不同颜色标识该行的变更类型，方便快速定位。

在 Diff 页面中，你可以通过 Sheet 标签页切换不同工作表。标签上带有 变更 标记的表示该 Sheet 有内容变化。

确认提交

管理提交记录和周版本文件的审阅状态

仓库提交确认

1

三种确认状态

每条仓库提交记录都有一个确认状态：

  • 待确认 — 提交尚未被审阅，为默认状态
  • 已确认 — 提交内容已被审阅并通过
  • 已拒绝 — 提交内容被审阅后拒绝
2

单个提交确认/拒绝

在提交记录列表中，每条提交右侧有操作按钮：

  • 点击 「✓ 通过」 按钮，将该提交标记为 已确认
  • 点击 「✗ 拒绝」 按钮，将该提交标记为 已拒绝
  • 已确认或已拒绝的提交，可以通过下拉菜单切换状态或重置为待确认
3

批量确认

在提交记录列表页面，可以勾选多条提交，然后点击 「批量确认」 按钮，一次性将所有选中的提交标记为已确认。

4

快速筛选

提交列表支持按确认状态筛选：

  • 点击 「未确认条目」 按钮可快速切换只显示待确认和已拒绝的记录
  • 也可以在筛选面板中选择特定状态进行精确过滤

周版本文件确认

1

文件级别确认

在周版本 Diff 页面中，每个 Excel 文件都有独立的确认状态。进入文件的 Diff 详情后，可以使用顶部的操作按钮：

  • 点击 「确认通过」 按钮，表示该文件的变更审阅通过
  • 点击 「拒绝」 按钮，表示该文件的变更需要修改
2

快捷操作

在文件列表中，每个文件右侧提供了快捷状态按钮，无需进入 Diff 详情即可直接确认或拒绝，提高审阅效率。

3

批量确认

点击周版本 Diff 页面顶部的 「批量确认」 按钮，可以一次性将所有待确认的文件标记为已确认，适用于整体审阅无异议的情况。

4

进度统计

周版本列表页面会显示每个配置的确认进度统计：

  • 待确认数：尚未审阅的文件数量
  • 已确认数：已通过审阅的文件数量
确认/拒绝操作仅用于标记审阅状态，不会影响实际的配表数据或仓库内容。修改周版本的时间范围会清空所有确认状态。
6

缓存管理

管理系统的 Diff 缓存数据，优化性能

缓存加速原理

系统会缓存已生成的 Diff 数据和 HTML 渲染结果，避免重复计算。首次查看 Diff 时可能较慢，后续访问将直接使用缓存，速度极快。

缓存清理

在 「缓存管理」 页面，你可以：

  • 清理过期：清理超时的普通缓存和已过期的长处理文件缓存
  • 清理旧版本：当 Diff 逻辑版本升级后，清理旧版本的缓存
  • 清理全部：删除所有缓存（下次访问需重新生成）
清理全部缓存后，所有 Diff 页面在首次访问时需要重新计算和渲染，可能需要较长时间，请谨慎操作。
