1

新建项目

创建一个新项目来管理你的配表仓库

1

进入项目管理页面

点击顶部导航栏的 「项目列表」 进入主页，点击 「新建项目」 按钮。

2

填写项目信息

在弹出的表单中填写以下信息：

  • 项目代号：项目的唯一标识符，如 G119，用于URL路径
  • 项目名称：项目的可读名称，如「某某项目」
  • 项目描述：（可选）简要描述项目用途
3

提交创建

确认信息无误后点击「创建」按钮，项目即创建成功。创建后可在项目卡片中进一步配置仓库。

项目代号创建后不可修改，请谨慎填写。建议使用简短的英文+数字组合。
2

配置 SVN / Git 仓库

为项目添加代码仓库以追踪配表变更

1

进入项目详情

在项目列表中点击对应项目的 「管理」 按钮，进入项目管理页面。

2

添加仓库

点击 「添加仓库」 按钮，选择仓库类型并填写信息：

Git 仓库
  • 仓库名称：仓库显示名
  • 仓库URL：Git 远程地址（HTTPS 或 SSH）
  • 默认分支：如 main 或 master
  • 认证方式：用户名密码或SSH密钥
SVN 仓库
  • 仓库名称：仓库显示名
  • 仓库URL：SVN 仓库地址
  • 用户名：SVN 登录用户名
  • 密码：SVN 登录密码
3

同步提交记录

仓库添加成功后，系统会自动开始同步提交记录。你也可以手动点击 「同步」 按钮来获取最新提交。

请确保仓库地址可访问，认证信息正确。如果使用SSH方式，请确认服务器已配置对应的SSH密钥。
3

周版本管理

按周管理配表版本，支持合并多次提交的 Diff

1

创建周版本配置

在项目管理页面，点击 「周版本管理」 标签页，然后点击 「新增周版本」。

  • 版本名称：如「2025年第12周」
  • 起止日期：设定该周版本覆盖的时间范围
  • 关联仓库：选择要追踪的仓库
2

查看周版本 Diff

周版本创建后，系统会自动合并该时间范围内所有提交的 Diff。点击 「查看Diff」 可以看到整个周期的合并变更。

周版本 Diff 会合并所有单次提交的变更，只展示最终的新旧值差异，便于整体审阅。
4

查看 Excel Diff

对比 Excel 配表文件的变更内容

1

进入仓库提交列表

在项目管理页面，点击对应仓库的 「提交记录」，查看所有提交历史。

2

查看单次提交 Diff

点击某次提交记录的 「查看Diff」 按钮，进入 Diff 详情页面。

  • 页面顶部显示提交的基本信息（提交者、时间、消息）
  • 如果提交包含 Excel 文件变更，将以表格形式显示差异
  • 多个 Sheet 页通过标签页切换查看
3

确认状态管理

每条提交记录都有确认状态，可标记为「待确认」「已确认」「已拒绝」，方便团队协作追踪。

Diff 颜色与标识说明

了解 Excel Diff 中各种颜色和标记的含义

行级别变更

1001 new_key 新增的值 true
新增行（整行绿色底色）

表示该行是新版本中新增的数据行，之前不存在。

502 old_key 已删除的值 false
删除行（整行红色底色）

表示该行在新版本中已被删除，只存在于旧版本中。

单元格级别变更

677 some_key 旧的值 新的值 true
修改的单元格

当某一行只有部分单元格发生变化时：

  • 红色背景 表示 旧值（修改前的值）
  • 绿色背景 表示 新值（修改后的值）
  • 未变化的单元格保持白色背景

行号标识

1001 新增行 502 删除行 677 修改行
行号颜色

行号列也会用不同颜色标识该行的变更类型，方便快速定位。

在 Diff 页面中，你可以通过 Sheet 标签页切换不同工作表。标签上带有 变更 标记的表示该 Sheet 有内容变化。
6

缓存管理

管理系统的 Diff 缓存数据，优化性能

缓存加速原理

系统会缓存已生成的 Diff 数据和 HTML 渲染结果，避免重复计算。首次查看 Diff 时可能较慢，后续访问将直接使用缓存，速度极快。

缓存清理

在 「缓存管理」 页面，你可以：

  • 清理过期：清理超时的普通缓存和已过期的长处理文件缓存
  • 清理旧版本：当 Diff 逻辑版本升级后，清理旧版本的缓存
  • 清理全部：删除所有缓存（下次访问需重新生成）
清理全部缓存后，所有 Diff 页面在首次访问时需要重新计算和渲染，可能需要较长时间，请谨慎操作。
